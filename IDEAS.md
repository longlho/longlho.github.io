# Ideas

1. Something around "metaframework is dead": that is consolidation at runtime. The future should be more build time, plus agents that can replicate certain setups. Verbose scaffolding is actually better; the default should be `eject` mode.
2. Trash file-based routing. Prefer explicit route registration. The authoring experience can still feel implicit, but the framework should codegen that into explicit route code. File names and path conventions create weird constraints, including charset conflicts with the OS and framework-specific naming rules that leak into the product structure.
